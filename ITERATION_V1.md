# 个人作品集项目 - 一期迭代总结 (V1.0)

## 1. 需求与功能变更 (Feature Delta)

- 新增了轻量后台编辑能力：当前代码实现了 `/admin` 管理入口、登录页与可视化编辑面板，可在线编辑 About、个人信息、社交链接、经历与项目，并支持新增/删除条目；该能力未在原 PRD 中定义。
- 新增了“本地持久化内容管理”业务逻辑：编辑数据会写入 `localStorage`，刷新后仍保留，同时提供“一键重置初始数据”能力；原 PRD 仅定义静态展示型作品集。
- 内容模型从“模块独立数据”演进为“统一 PortfolioData”：`about/aboutKeywords/profile/socials/experiences/projects` 被整合到单一数据对象，驱动首页与后台共享同一数据源。
- “关于我”区块新增关键词高亮机制：通过 `aboutKeywords` 进行文本匹配并动态强调关键技术词，属于原 PRD “可强调关键词”的实现细化，但数据结构与实现方式为新增。
- 导航交互细化为“平滑滚动 + URL Hash 同步”：点击侧边栏会平滑滚动并更新地址栏锚点，较 PRD 的“滚动监听高亮”增加了可分享定位体验。
- 社交链接由固定图标扩展为可配置图标名：`SocialLink` 引入 `iconName` 字段并通过映射渲染图标，提升了可扩展性，但 PRD 未定义该字段。
- 404 页当前仅完成基础占位版本：存在独立 404 路由与返回首页入口，但“终端风格 + 打字机 + 故障特效”等 PRD 定义的完整体验尚未落地，属于需求收敛中的阶段性偏差。

## 2. 技术实现与架构演进 (Tech Updates)

- 路由架构发生扩展：除首页与通配 404 外，新增 `/admin` 页面，形成“展示端 + 编辑端”双场景路由结构，超出初始 TECH_DESIGN 的单一展示站点范围。
- 数据层从“分文件静态数据”改为“集中式内容源”：实际使用 `src/data/content.ts` 与 `PortfolioData` 聚合模型，而非设计稿中的 `experience.ts`/`projects.ts` 分拆方式。
- 状态管理策略由“纯静态渲染”升级为“运行时数据读取与规范化”：`usePortfolioData` 负责初始化、兼容旧数据结构、保存与重置，体现了内容演进过程中的向后兼容设计。
- 动画实现路线与设计预期存在偏差：项目未引入 Framer Motion，当前主要使用 CSS Transition + 原生 API（IntersectionObserver、scrollIntoView、鼠标事件）完成交互。
- Scrollspy 实现方案增强：不仅监听可见性，还按“视口中心最近区块”计算激活态，解决多区块同时可见时的高亮抖动问题，较初版技术方案更稳定。
- Spotlight 细节增强：鼠标 Hook 初始化为视口中心，避免首屏光晕锁定左上角的视觉瑕疵，属于体验层面的工程优化。
- 管理端认证采用前端会话态方案：使用硬编码口令 + `sessionStorage` 记录登录状态，仅适用于本地演示/个人站场景，和生产级鉴权（服务端校验）存在明确边界。

## 3. 开发规范沉淀 (Rules Refinement)

- 统一内容模型规则：所有展示内容必须先收敛到 `PortfolioData` 类型，再由组件消费，禁止在组件内新增临时字段或绕过类型直接读写 `localStorage`。
- 交互一致性规则：导航、滚动高亮、卡片 Hover、Spotlight 等体验必须优先复用现有 Hook/样式模式；新增交互前先评估是否可扩展现有实现，避免重复造轮子。
- 渐进增强规则：404、后台等非核心路径允许分阶段交付，但每个阶段都需保留可用兜底（可访问、可返回、可读提示），并在文档中标注“占位/未完成项”。
- 前端持久化边界规则：`localStorage/sessionStorage` 仅用于个人站本地配置与演示态，不承载敏感信息；涉及鉴权、密码、用户数据时必须迁移到服务端方案。
- 数据编辑可回滚规则：任何可视化编辑能力都必须提供“保存前本地态隔离 + 明确保存动作 + 一键重置”三件套，确保误操作可恢复、状态变更可追踪。
