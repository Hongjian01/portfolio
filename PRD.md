# 产品需求文档 (PRD)：开发者美学个人作品集网站

## 1. 产品概述
* **产品名称**：[你的名字] 个人作品集网站 (Portfolio Website)
* **产品目标**：打造一个极简、现代、沉浸式的个人网站，通过“IDE 级”的深色美学和像素级的微交互，高效且优雅地展示开发者的工程能力、硬核技术栈以及专业素养。
* **目标用户**：技术招聘方（HR/猎头）、技术面试官、潜在合作客户、开源社区同行。
* **核心设计理念**：少即是多 (Less is More)、性能至上、无障碍 (Accessibility) 优先。

## 2. 整体设计与体验规范 (Design & UX Guidelines)
本网站需严格遵循以下视觉与交互规范，以确保“极客专业感”的精准传达：

### 2.1 视觉风格 (Visual Style)
* **色彩系统**：
    * **主背景色**：深蓝或深石板灰（参考代码编辑器的主题底色，如 `#0f172a`）。
    * **文本色**：浅灰/银色作为主文本色，白色作为高亮标题色。
    * **强调色 (Accent Color)**：选取一种高饱和度的颜色（如亮青色、翠绿色或荧光蓝），用于链接、Hover 状态和核心按钮，模拟“语法高亮”。
* **排版 (Typography)**：
    * 全局使用单一的现代无衬线字体（强烈建议使用 **Inter** 或 **Roboto**）。
    * 严格通过**字号、字重（Light/Regular/Bold）和颜色透明度**来构建视觉层级，避免使用斜体或过多颜色。

### 2.2 核心交互 (Core Interactions)
* **全局鼠标聚光灯 (Spotlight Effect)**：在桌面端，背景需实现一个随鼠标指针移动的柔和高光/光晕效果，增加页面的空间纵深感。
* **滚动监听 (Scrollspy)**：左侧导航栏需实时监听右侧内容的滚动位置，并动态高亮当前所在的阅读章节。
* **卡片微交互 (Micro-interactions)**：鼠标悬停在“经历”或“项目”卡片时，触发平滑的过渡动画：
    * 背景区块轻微提亮。
    * 主标题文字颜色渐变为强调色。
    * 外链箭头图标产生向右上角的微小位移。

## 3. 核心功能模块 (Functional Requirements)

网站采用**单页应用 (SPA) 架构**。桌面端为“左固定、右滚动”的非对称双栏布局；移动端降级为从上到下的单栏瀑布流布局。

### 3.1 左侧边栏：个人品牌与导航 (Sidebar / Hero)
这是用户进入网站的第一印象，需保持固定可见（桌面端）。
* **核心信息展示**：
    * H1 级别的大字号姓名。
    * 当前职位或专业头衔（如 "Senior Backend Engineer" 或 "CS Graduate Student"）。
    * 一句简短有力的个人使命陈述（1-2 行）。
* **页面内导航 (Page Navigation)**：包含 About, Experience, Projects 等锚点链接。
* **社交网络矩阵**：底部放置干净的纯色 Icon 链接（GitHub, LinkedIn, 个人邮箱等），Hover 时变为强调色。

### 3.2 右侧主内容区：详细信息 (Main Content)
分为三个核心纵向滚动区块：

#### 区块一：关于我 (About)
* **内容要求**：2-3 段精炼的文字，描述职业背景、技术专长和对工程理念的追求。
* **呈现方式**：纯文本展示，可将核心关键词（如特定技术栈或算法理念）加粗或使用强调色。

#### 区块二：工作与实践经历 (Experience)
* **布局方式**：隐藏边界的“隐形卡片”列表，按时间倒序排列。
* **字段定义**：
    * **时间区间**：放置在左侧或上方小字号展示。
    * **职位与公司/实验室**：作为卡片标题，包含跳转外链。
    * **职责描述**：简明扼要的描述工作成果（突出业务价值和技术难点突破）。
    * **技术栈标签 (Tech Stack Pills)**：该经历中使用的核心技术，采用圆角药丸状 (Pill) 标签展示，文字使用强调色，背景为强调色的低透明度版本。

#### 区块三：重点项目展示 (Projects)
* **布局方式**：与经历区块类似的卡片列表，聚焦于硬核工程或科研项目。
* **字段定义**：
    * **项目名称**：点击直接跳转至项目演示地址、GitHub 仓库或论文链接。
    * **项目描述**：说明该项目解决的问题及核心亮点（例如：分布式架构优化、底层网络协议仿真等）。
    * **技术栈标签**：同工作经历区块。

### 3.3 边界情况与 404 错误页设计 (Edge Cases & 404 Page)
* **设计目标**：将枯燥的“页面未找到”异常转化为展现个人技术背景与极客幽默的微交互体验，同时提供明确的返回路径。
* **视觉与交互规范**：
    * 保持全站深色 IDE 沉浸感，去除所有复杂的装饰元素。
    * 全局强制切换为**等宽字体 (Monospace)**（如 JetBrains Mono），模拟真实的控制台/终端输出。
    * 引入模拟终端打字机逐字输出效果 (Typewriter Effect) 与光标闪烁动画。
* **内容策略（文案示例）**：
    * **标题 (H1)**：`404`（可带有轻微的毛刺故障特效）。
    * **错误日志输出**：`Exception in thread "main" java.lang.NullPointerException: Requested page '/unknown-path' is null.`
    * **交互出口 (CTA)**：设计成命令行输入样式（例如：`$ cd /home` 或 `↵ Return to root`）。Hover 时产生高亮反色，点击后平滑路由回网站首页。

## 4. 技术实现、部署与非功能性要求 (Technical & Deployment Requirements)

* **核心技术架构**：
    * 必须基于 **React 18 + Vite + TypeScript** 体系构建，确保组件化开发与类型安全。
    * 样式方案：推荐使用 **Tailwind CSS**，利用其实用类快速实现深色模式和响应式布局。
    * 动画方案：推荐引入 **Framer Motion** 处理平滑滚动监听、卡片 Hover 微交互与入场动画。
* **Vercel 部署与配置**：
    * 网站代码托管至 GitHub，并通过 Vercel 进行自动化 CI/CD 部署。
    * **SPA 路由重写（关键）**：必须在项目根目录创建 `vercel.json` 文件并配置路由重写规则，防止用户直接访问子路由或刷新页面时触发服务端 404 错误：
      ```json
      {
        "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }]
      }
      ```
* **性能与监控 (Performance)**：
    * 页面必须做到秒开，核心 Web 指标（LCP）需控制在 2.5s 以内，无明显的加载白屏。
    * 建议在 Vite 配置中对静态资源（图片、字体）进行合理的打包优化。
    * 建议集成 `@vercel/analytics` 和 `@vercel/speed-insights`，用于持续监控网站的真实访问流量和性能指标。
* **无障碍访问 (Accessibility - A11Y) - 【必须满足】**：
    * **全键盘导航支持**：必须支持仅用 `Tab` 键顺畅浏览全站所有可交互元素。
    * **Focus 状态**：处于焦点状态的元素必须有清晰可见的轮廓线 (Outline)。
    * **语义化 HTML**：正确使用 `<header>`, `<main>`, `<section>`, `<article>`, `aria-labels`。
* **响应式适配 (Responsive Design)**：
    * 断点设计：兼容 Mobile (竖屏单栏), Tablet (窄屏), Desktop (双栏宽屏)。
    * 在移动端 (Mobile) 下，原先左侧固定的内容需折叠为页面的头部区块 (Hero Section)，右侧内容顺次变为下方滚动的信息流。
